\begin{abstract}
Scalable automated grading of handwritten answers presents a significant challenge in educational technology. We present the Phase 1 non-neural pipeline of our Automated EdTech Grading Assistant, which combines classical Tesseract OCR with statistical natural language processing. Using the SciEntsBank dataset, evaluated as a 3-way classification task, we establish a reproducible baseline. Our approach isolates the performance of legacy OCR and term frequency-based text matching prior to introducing deep learning techniques. This baseline intentionally uses non-neural components to establish an interpretable lower bound. The pipeline achieves an F1 Macro score of \textbf{0.4974} on unseen domains, highlighting the limitations of classical lexical matching.
\end{abstract}

\section{Introduction}
Manual grading of student assessments does not scale efficiently beyond thirty students, making automated grading a critical requirement for modern educational technology at scale. However, transitioning from typed to handwritten input introduces novel challenges. 

Challenge 1 is handwriting variability. Legacy OCR engines like Tesseract struggle significantly with cursive and degraded student writing. Challenge 2 is the semantic gap. Classical lexical overlap metrics (such as Jaccard similarity) fail to capture meaning when students use synonyms or restructure sentences. Challenge 3 is the inherent class imbalance in datasets like SciEntsBank, where correct answers often dominate, complicating supervised learning.

What this paper does: It establishes an end-to-end classical OCR and statistical NLP pipeline. What this paper does NOT do: It intentionally avoids neural methods (which are reserved for Phase 2). This baseline intentionally uses non-neural components to establish an interpretable lower bound, setting the stage for subsequent ablation studies.

\section{Related Work}
Dzikovska et al. (2013) introduced the SciEntsBank dataset in ``Semeval-2013 task 7: The joint student response analysis and 8th recognizing textual entailment challenge" \cite{dzikovska2013}. This benchmark provides a standardized platform for assessing science education answers.

Classical text matching approaches have historically relied on statistical term frequencies. Robertson \& Zaragoza (2009) established the origins of BM25 in ``The Probabilistic Relevance Framework: BM25 and Beyond" \cite{robertson2009}. Similarly, Salton \& Buckley (1988) popularized TF-IDF in ``Term-weighting approaches in automatic text retrieval" \cite{salton1988}. While effective for search, these methods fall short on semantic inference. Mohler \& Mihalcea (2009) explored these limitations in ``Text-to-text semantic similarity for automatic short answer grading" \cite{mohler2009}, highlighting the gap in handling synonymous terminology. 

In recent years, neural short answer grading has advanced rapidly, such as the work by Sultan et al. (2016) \cite{sultan2016}. However, the SciEntsBank benchmark highlights the critical need to test models under distribution shifts (e.g., unseen domains). 

Gap: No existing work combines OCR and semantic grading in a unified pipeline for the specific nuances of handwritten science answers. Existing systems assume typed text input. No prior work addresses the handwritten $\rightarrow$ graded pipeline end-to-end, which is the practical classroom scenario.

\section{Methodology}
\subsection{Dataset}
The SciEntsBank dataset comprises student answers across diverse science domains. The splits include training data, unseen answers (test\_ua), unseen questions (test\_uq), and unseen domains (test\_ud). Analysing the label distribution reveals a pronounced class imbalance (as shown in Section 4). 

We collapse the original 5-way labels into a 3-way classification: correct (0), contradictory (1), and partially correct (2). Collapsing irrelevant, non\_domain, and partially\_correct into class 2 is motivated by pedagogical equivalence — all three require teacher intervention.

The train and test splits follow a semantic strategy. UA tests generalization to new phrasings, UQ checks performance on new prompt types within known topics, and UD is the strictly out-of-distribution test for entirely new science domains.

\subsection{OCR Pipeline}
For transcription, we utilize the Tesseract architecture via its legacy LSTM-free legacy engine (\texttt{--oem 0}). We implement robust preprocessing to optimize recognition: Grayscale reduces 3-channel noise to 1-channel signal; Autocontrast normalizes illumination variance; Binarization converts pixels to $\{0, 255\}$ for Otsu-like thresholding.

We intentionally use the non-neural engine to maintain a pure classical baseline. \texttt{--oem 3} uses an LSTM internally, which would contaminate our Phase 1 non-neural claim. Known limitations include poor performance on cursive handwriting and low-resolution student scans.

\subsection{Feature Engineering}
The model utilizes a suite of statistical and lexical features.

TF-IDF captures term importance:
\begin{equation}
\text{tfidf}(t,d) = \text{tf}(t,d) \cdot \log\frac{N}{\text{df}(t)}
\end{equation}

Jaccard similarity measures exact set overlap:
\begin{equation}
J(A,B) = \frac{|A \cap B|}{|A \cup B|}
\end{equation}
Jaccard is symmetric and set-based. It cannot capture semantic equivalence between synonyms (e.g., 'energy' and 'ATP'). This motivates Phase 2.

Token Density ($\delta$) calculates relative overlap:
\begin{equation}
\delta = \frac{|S \cap R|}{|R|}
\end{equation}
where $S$ is the student answer and $R$ is the reference answer.

Length features include absolute counts and averages: $[|s|, |W_s|, \bar{w}]$.

\subsection{Classifier}
We deploy a Logistic Regression classifier:
\begin{equation}
P(y=k|x) = \frac{e^{w_k^T x}}{\sum_j e^{w_j^T x}}
\end{equation}
To counteract class imbalance, we use inverse frequency weighting (\texttt{class\_weight=balanced}):
\begin{equation}
w_k = \frac{n}{K \cdot n_k}
\end{equation}
where $n$ is total samples, $K$ is the number of classes, and $n_k$ is class count. We optimize via the L-BFGS solver with standard convergence criteria. LogReg provides probability estimates natively, enabling calibrated confidence scores. SVM requires Platt scaling. For the baseline we prefer simplicity.

\subsection{Error Analysis}
The classifier struggles most with the boundary between 'correct' and 'partially correct', primarily because partial credit answers often share significant lexical overlap with the reference, deceiving the Jaccard logic. 

There is a significant performance gap across the test sets constraint (UA vs. UQ vs. UD). Specifically, UD performance drops because TF-IDF vocabulary learned during training does not transfer to unseen domains.

\section{Discussion}
This pipeline successfully establishes our non-neural baseline. However, its failures act as designed experiments mapping the limitations of classical techniques. The 3 specific limitations that motivate Phase 2:
\begin{enumerate}
    \item Jaccard cannot handle synonyms $\rightarrow$ need SBERT
    \item Tesseract fails on cursive $\rightarrow$ need TrOCR
    \item TF-IDF fails on UD split $\rightarrow$ need semantic embeddings
\end{enumerate}
These are not failures — they are designed experiments.

\section{Conclusion}
We have instituted a reproducible, fully classical evaluation baseline pairing Tesseract OCR and Logistic Regression against the SciEntsBank dataset. Phase 2 of this project will dismantle these specific limitations by implementing a neural pipeline layered onto a semantic-aware classifier.
