The Challenge of Automation

Handwriting Variability

Student handwriting across scans varies in pressure, style, and cursive соединение. Legacy OCR suffers from high Character Error Rates (CER).

Semantic Gap

Keywords alone (TF-IDF) can't distinguish between "heat increases" and "energy goes up" without heavy synonym mapping.

The Lexical Trap

Classical models relying on Jaccard overlap are easily "fooled" by long, irrelevant answers that happen to contain many reference keywords (The Keyword Hallucination problem).

Implementation

Phase 1: Classical Baseline

Pipeline Architecture

  • 🔹 OCR: Tesseract Legacy Engine (OEM 0)
  • 🔹 Pre-processing: Grayscale + Otsu Binarization
  • 🔹 Features: TF-IDF N-grams + Jaccard Similarity
  • 🔹 Model: Logistic Regression (L-BFGS)
  • 🔹 Class Weights: Balanced Inverse Sampling

Tuning the Hybrid

Decision Boundary Analysis

Alpha (α) represents the weight given to the Classical Model (SVM). In our UA split calibration, we observed that high α favors precision on highly familiar questions, while lower α (relying on SBERT) is required for out-of-distribution stability.

Phase 3: Explainability

The "Why" Layer

Moving beyond black-box classification to pedagogical feedback. Phase 3 will introduce:

  • 📝 Noun-Chunk Mapping: Highlighting exactly which concepts (e.g., "evaporation") were missed.
  • 🔍 SHAP/LIME: Visualizing pixel-level importance for OCR and token-level for grading.
  • 💬 LLM Feedback: Generating human-like coaching comments based on hybrid scores.
